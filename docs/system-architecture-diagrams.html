9. ER図（データベース）

データベーステーブルのリレーションとフィールド定義です。

テーブル説明

テーブル名 説明 主要カラム
users ユーザー（求職者・企業） role で seeker/employer を区別。両方のフィールドを含む非正規化構成
jobs 求人情報 embedding フィールドでベクトル検索用の埋め込みを保存（1536次元）
applications 応募履歴 求職者が求人に応募した記録。status で選考状況を管理
scouts スカウト 企業から求職者への求人オファー。status で既読・返信状況を管理

10. デプロイメントフロー図

GitHub ActionsによるCI/CDパイプラインのフローです。

CI/CDパイプライン詳細

ステップ 処理内容 ツール/コマンド
バックエンド
1. テスト Pytestでユニットテスト・統合テスト実行 pytest app/tests/
2. Docker Build Dockerfileからコンテナイメージをビルド docker build -t job-matching-backend .
3. ACRプッシュ Azure Container Registryにプッシュ docker push acr.azurecr.io/backend:latest
4. デプロイ Azure App Serviceにデプロイ az webapp deploy --resource-group ...
5. ヘルスチェック /health エンドポイントで正常性確認 curl https://backend.azurewebsites.net/health
6. ロールバック 失敗時に前バージョンへ自動復元 az webapp deployment slot swap
フロントエンド
1. テスト Vitestでコンポーネントテスト実行 npm run test
2. ビルド Viteでプロダクションビルド npm run build
3. デプロイ Azure Static Web Appsにデプロイ Azure SWA CLI / GitHub Actions
4. CDN無効化 Azure Front DoorのCDNキャッシュクリア az cdn endpoint purge
