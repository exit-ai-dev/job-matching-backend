1. システム概要・スコープ

1.1 プロジェクト概要

本ドキュメントは、求人マッチングプラットフォームの「Webアプリケーション部分」の設計を定義する。 AIの内部ロジック（プロンプト等）およびインフラ詳細（Azure設定等）は別ドキュメントにて定義する。

1.2 主要機能

  • 求職者機能: プロフィール登録、求人検索、AIチャット、応募管理
  • 企業機能: 求人作成・管理、候補者検索、スカウト送信
  • AI機能: ベクトル埋め込みによるマッチング、自然言語での求人検索
  • LINE連携: LINE LIFF経由の認証とプッシュ通知

1.3 修正・変更点 (v2.0→v2.1)

  • 実装構成の反映: 実際のファイル構成、DB設計を正確に記載
  • 詳細仕様の追加: バリデーション、エラーハンドリング、環境変数の詳細を追加
  • 画面一覧の追加: 全13画面の遷移図と詳細仕様を追加
  • 外部サービス: OpenAI API、LINE LIFF連携の詳細を追加

7. 認証・セキュリティ設計

7.1 トークン管理

種別保存場所有効期限
Access Tokenメモリ (React State)15分
Refresh TokenHttpOnly Cookie7日

ローテーション: Refresh Token使用時に新しいRefresh Tokenを発行し、使用済みトークンを無効化。

7.2 RBAC (Role-Based Access Control)

RolePermissions
seeker求人閲覧、応募、プロフィール編集
employer求人作成、候補者検索、スカウト送信
admin全データ閲覧・削除、ユーザーBAN

7.3 セキュリティ対策

項目対策実装箇所
パスワード保護 bcrypt ハッシュ化 (cost=12) app/api/endpoints/auth.py
CORS 許可オリジンを環境変数で制限 (credentials: true) app/main.py, app/core/config.py
SQLインジェクション SQLAlchemy ORM使用、Parameterized Query 全models/
XSS Reactのデフォルトエスケープ、dangerouslySetInnerHTML禁止 全frontend/
CSRF JWTトークンベース認証、SameSite Cookie app/core/dependencies.py
Rate Limiting FastAPI SlowAPI (100 req/min/IP) app/main.py (middleware)
入力検証 Pydantic バリデーション (全API) app/schemas/

10. AI機能連携仕様

10.1 ベクトル埋め込み処理

使用モデル: OpenAI text-embedding-3-small (1536次元)
対象データ: 求人description + title、ユーザーskills + experience
# app/ml/embedding_service.py def generate_embedding(text: str) -> List[float]: """ テキストをベクトル埋め込みに変換 """ response = openai.Embedding.create( model="text-embedding-3-small", input=text ) return response['data'][0]['embedding'] # 1536次元のベクトル

10.2 マッチングスコア計算

コサイン類似度を0-100%のスコアに変換

# app/ml/matching_service.py from sklearn.metrics.pairwise import cosine_similarity def calculate_match_score(seeker_embedding, job_embedding) -> float: """ ベクトル間のコサイン類似度を計算し、0-100のスコアに変換 """ similarity = cosine_similarity([seeker_embedding], [job_embedding])[0][0] # コサイン類似度: -1 〜 1 → 0 〜 100% に正規化 score = (similarity + 1) / 2 * 100 return round(score, 2)
