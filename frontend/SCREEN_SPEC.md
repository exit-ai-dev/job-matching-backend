# exitotrinity - 画面構成仕様書

## 1. プロジェクト概要

**プロジェクト名**: exitotrinity（エグジトトリニティ）
**種別**: AI求人マッチングプラットフォーム
**対象ユーザー**: 求職者（seeker）と企業（employer）

---

## 2. 画面一覧

| 画面名 | URL | ログイン | 対象ユーザー | 説明 |
|--------|-----|----------|--------------|------|
| LandingPage | `/` | 不要 | 全員 | サービス紹介ページ |
| LoginPage | `/login` | 不要 | 全員 | ログイン画面 |
| RegisterPage | `/register` | 不要 | 全員 | 新規登録画面 |
| LineLinkPage | `/auth/line-link` | 必要 | 全員 | LINE連携画面 |
| PreferencesPage | `/preferences` | 必要 | 求職者のみ | 希望条件入力画面 |
| HomePage | `/home` | 必要 | 全員 | ダッシュボード |
| ChatPage | `/chat` | 必要 | 全員 | AI検索・会話画面 |

---

## 3. 画面遷移フロー

### 3.1 新規登録フロー

```
LandingPage
  ↓ 「無料で始める」クリック
RegisterPage
  ↓ 登録完了
LineLinkPage
  ↓ LINE連携完了
[求職者の場合] → PreferencesPage → HomePage
[企業の場合] → HomePage
```

### 3.2 ログインフロー

```
LandingPage / LoginPage
  ↓ ログイン
HomePage
```

## 5. データモデル（画面表示用）

### 5.1 ユーザー（User）

```typescript
interface User {
  id: string;
  email: string;
  name: string;
  role: 'seeker' | 'employer';  // 求職者 or 企業
  lineLinked?: boolean;
}
```

### 5.2 求人（Job）

```typescript
interface Job {
  id: number;
  title: string;              // 職種名
  company: string;            // 企業名
  salary: string;             // 年収範囲
  location: string;           // 勤務地
  matchScore: number;         // マッチ度（0-100）
  tags: string[];             // タグ（技術スタック等）
  type: string;               // 雇用形態
}
```

### 5.3 候補者（Candidate）

```typescript
interface Candidate {
  id: number;
  name: string;               // 名前
  age: number;                // 年齢
  position: string;           // 希望ポジション
  experience: string;         // 経験年数
  matchScore: number;         // マッチ度（0-100）
  skills: string[];           // スキル
  location: string;           // 居住地
  status: string;             // 選考ステータス
}
```

### 5.4 メッセージ（Message）

```typescript
interface Message {
  id: string;
  role: 'user' | 'assistant';  // ユーザー or AI
  content: string;             // メッセージ本文
  timestamp: Date;             // 送信時刻
}
```

### 5.5 検索結果（SearchResult）

```typescript
interface SearchResult {
  id: number;
  title: string;              // 職種名 or 候補者名
  company?: string;           // 企業名（求人の場合）
  name?: string;              // 名前（候補者の場合）
  matchScore: number;         // マッチ度
  details: string[];          // 詳細情報（配列）
  tags: string[];             // タグ
}
```

## 6. 画面間データ連携

### 6.1 認証フロー

1. RegisterPage で入力されたデータ:
   - email, password, name, role
   - → API送信 → トークン取得 → localStorage保存

2. LoginPage で入力されたデータ:
   - email, password
   - → API送信 → トークン取得 → localStorage保存

3. 認証後、全画面でユーザー情報を使用

### 6.2 希望条件フロー

1. PreferencesPage で入力されたデータ:
   - 希望年収（salary）
   - 希望職種（jobType）
   - 重視する条件（preferences, 配列）
   - → API送信 → HomePage へ遷移

### 6.3 検索フロー

1. ChatPage でユーザーが条件を入力:
   - メッセージ内容
   - → API送信 → AI応答 + 検索結果取得
   - → 右カラムに結果表示

## 7. エラー処理

### 7.1 表示するエラー

- **ログイン失敗**: 「メールアドレスまたはパスワードが正しくありません」
- **登録失敗（メール重複）**: 「このメールアドレスは既に登録されています」
- **登録失敗（バリデーション）**: 「パスワードは8文字以上で入力してください」
- **ネットワークエラー**: 「通信に失敗しました。しばらくしてから再度お試しください」

### 7.2 エラー表示位置

- フォーム上部に赤枠のアラートボックスで表示
- エラーメッセージはシンプルで具体的に

## 8. ローディング状態

### 8.1 表示するローディング

- ボタン押下後: ボタンテキストを「ログイン中...」「登録中...」「検索中...」に変更

### 8.2 ChatPage のローディング

- AI応答待機中: メッセージエリアに「検索中...」メッセージを表示
